    微信渠道接口文档

    消息平台文档管理系统 2019-06-01 渠道 接口

    # 1. 功能概述:

    • 功能概述 微信渠道支持单发、批量推送、群发、分组推送
      • 单发(客服消息) 推送给单个人,可推送类型为:普通文本消息、图片消息、音频消息、视频消息、音乐消息、图文消息(点击跳转到外链)、图文消息(点击跳转到图文消息页面)、模版消息(不需要用户处于活跃状态) 注:用户主动与微信公众号交互的48小时之内，可以使用该消息类型向该用户进行消息推送(模版消息除外)
      • 批量推送(客服消息) 推送给非标签成员的无交集的一批人员,可推送类型为:普通文本消息、图片消息、音频消息、视频消息、音乐消息、图文消息(点击跳转到外链)、图文消息(点击跳转到图文消息页面)、模版消息(不需要用户处于活跃状态) 批量支持不同用户同时推送不同类型不同内容的消息 注:用户主动与微信公众号交互的48小时之内，可以使用该消息类型向该用户进行消息推送(模版消息除外)
      • 群发 群发分为根据openId集合群发和发全部人员 推送给拥有同一标签的一批人员,可推送类型为:普通文本消息、图片消息、语音(音频)消息、视频消息、图文消息
      • 分组推送 推送当前微信公众号的全部人员,可推送类型为:普通文本消息、图片消息、语音(音频)消息、视频消息、图文消息

    # 2. 接入流程:

    1. 新增接入系统,若其他渠道已新增不必新增,新增时必须选定需要发送的渠道,没有选择则没有该渠道发送权限
    2. 新增微信配置,"渠道管理-微信配置",配置微信发送appid和appsecret
    3. 系统绑定微信配置,"接入管理-系统绑定",若未绑定微信配置,则查询默认微信配置,都没有不能推送消息

    # 3. 接口说明:

    1. 微信发送统一调用消息发送接口，消息发送类型(sendType) 为 "2"
    2. 消息类型(messageType):
      • 文本消息:0
      • 视频消息:1
      • 图片消息:2
      • 图文消息:3
      • 音乐消息:4
      • 音频消息:5
      • 卡券消息:6(暂不支持)
      • 模版消息:7
    3. 微信推送给指定单个人或根据openId群发，需要当前用户对于微信公众号的唯一标识openId，若进行过用户绑定(接入管理--用户绑定)，请确保openId已与用户绑定，并传入用户id
    4. 推送图文消息时(跳转外部链接除外),图文内容中的图片不能写外部链接,应指定内容图片名称,进行文件的上传方可使用

    # 4. 调用说明:

    1. 单发
    2. 批量发不同消息
    3. 群发相同消息
    4. 分组推送相同消息

      当articleType为2时，为跳转到图文消息页面

      description 图文描述 String 否 图文消息描述

      详细介绍messageContent中的参数

      请求参数 参数名称 参数类型 是否必填 详细说明
      media_id
      多媒体文件id
      		String 否 传入时，下面的参数失效
      articles 图文消息 JsonArray 否 未传入media_id时生效,只能传入一条图文消息
      thumbImageName 缩略图文件名 String 是 缩略图文件名,对应上传文件中的图片
      contentImageNames 内容中的图片 JsonArray 否 内容中的图片,对应上传文件中的图片
      author 作者 String 否 图文消息的作者
      title 标题 String 是 图文消息的标题
      content_source_url 阅读原文的连接 String 否 点击阅读原文的跳转的链接
      content 图文消息内容 String 是 图文消息页面的内容，支持HTML标签,若内容中有图片并且时,在需要图片的位置加入<img src = ${文件名}}>,文件名为对应到contentImageNames中的文件名
      digest 图文消息描述 String 否 图文消息的描述，如本字段为空，则默认抓取正文前64个字
      show_cover_pic 是否显示封面 int 否 是否显示封面在内容的开头处，1为显示，0为不显示
      need_open_comment 是否打开评论 int 否 是否打开评论，0不打开，1打开
      only_fans_can_comment 是否粉丝才可评论 int 否 是否粉丝才可评论，0所有人可评论，1粉丝才可评论
