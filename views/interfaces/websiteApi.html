# 站内消息发送说明

# 1. 功能概述

  • 站内消息推送为接入系统提供站内消息发送以及接收能力
  • 支持跨系统消息推送功能
  • 可通过websocket方式接收消息
  • 可通过ajax轮询方式接收消息

# 2. 接入流程

  • weboscket接入流程

    • 接入服务器
      • 前端需配置websocket服务地址
        • 示例
            const WSURL = "ws://localhost:8081/git/websocket";
      • 当连接成功后发送一条包含系统编码和当前用户id的消息到websocket服务，以绑定当前用户
        • 示例
            const SYSTEMCODE = "PORTAL";
    const USERID = parent.indexTool.getUserInfo().userId;
    let actions = {};
    actions.systemCode_userId = SYSTEMCODE + "_" + USERID;
    this.websocket.send(JSON.stringify(actions));
        注：消息内容的key值必须为"systemCode_userId"
    • 接收消息
      • 接收消息参数
        • 示例
            {
    	"messageContent": "站内消息推送示例-内容：321",
    	"messageTitle": "站内消息推送示例-标题：123",
    	"sendUserId": "1"
    }
      • 示例
          // 数据接收
    websocketonmessage(msgData) {
        if (msgData && msgData.data) {
            let msg = JSON.parse(msgData.data);
            this.$message.success(msg.messageContent);
            let messageInfo = "接收到用户id为" + msg.sendUserId + "发送的消息，消息标题：" + msg.messageTitle +"　消息内容：" + msg.messageContent;
            this.list.push(messageInfo);
            console.log(msg);
        }
    }

  • ajax轮询接入说明

    • 调用查询地址
      • 示例
           http://localhost:8081/git/api/webSiteMessage/queryLatestNotReadWebSiteMessage
      • 请求方式 GET
    • 调用查询地址传入参数
      • rsystemCode 接收系统编码
      • receiveUserid 接收用户id
    • 返回值
      • 示例
          {
    	"msg": "查询成功！",
    	"result": {
    		"id": "24a8cc16c7454dcd9c11e4c37e23a883",
    		"receiveUserid": "1",
    		"rsystemCode": "PORTAL",
    		"messageTitle": "站内消息推送示例-标题：123",
    		"messageContent": "站内消息推送示例-内容：321",
    		"read": "0",
    		"sendUserId": "1"
    	},
    	"code": "200",
    	"requestId": "eecf171b-e0bc-4206-b061-df32fb448d42",
    	"success": true
    }
    • 查询方法
      • 示例
          fnQueryLatestNotReadWebSiteMessage(){
        let vue = this;
        //解决axios请求传入参数是json问题
        let data= new URLSearchParams();
        data.append('rsystemCode',SYSTEMCODE);
        data.append('receiveUserid',USERID);
        Axios({
            method: 'post',
            url: faceConfig.basePath + "/api/webSiteMessage/queryLatestNotReadWebSiteMessage",
            withCredentials:true,//携带cookies
            params: data
        }).then((response) => {
            let resultInfo = response.data;
            if(resultInfo.code === "200"){
                if(resultInfo.result != null){
                    //当此消息第一次被查询出来时弹出，同时记录此消息id
                    if(vue.messageId !== resultInfo.result.id){
                        vue.$message.success(resultInfo.result.messageContent);
                        vue.messageId = resultInfo.result.id;
                    }
                }
            }
        });
    }
      注：后台查询出来的始终是最近一次的未读消息，所以如果不想每一次该条消息都展示出来，需要进行处理
    • 轮询接收消息（10秒查询一次）
      • 示例
          this.myInterval = window.setInterval(() => {
      setTimeout(() => {
        this.fnQueryLatestNotReadWebSiteMessage();
      }, 1)
    }, 10000);

  • 发送消息

    1. 新增接入系统，若其他渠道已新增不必新增，新增时必须选定需要发送的渠道，没有选择则没有该渠道发送权限
    2. 按照消息发送api提供消息主体和消息子体中所需内容，再调用接口即可

# 3. 接口说明

  1. 站内消息发送统一调用消息发送接口，消息发送类型（sendType）为 “0”
  2. 站内消息无群发、分组发送，指定无效

# 4. 参数说明

  • receiveUserid 接收人id
  • rsystemCode 接收系统编码 注：同一批次消息的接收人和接收系统不可同时相同
