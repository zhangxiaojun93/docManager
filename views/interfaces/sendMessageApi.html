    消息发送接口文档

    消息平台文档管理系统 2019-07-20 消息发送 接口

    # 1. 接口描述

    • 接口名称 api/message/sendMessage 消息发送接口

      调用地址示例

      http://localhost:8081/git/api/message/sendMessage
      （消息平台地址 + api/message/sendMessage）

    • 接口编码
      UTF-8
    • 数据格式 JSON
    • 支持协议 HTTP(POST)

    # 2. 请求参数

    • 主体参数
    请求参数 参数类型 参数说明 是否必选
    sendUserid String 发送人员ID 是
    sendName String 发送人员姓名 是
    ssystemCode String 发送系统编码 是 [1]
    ticket String 系统秘钥 是
    messageTitle String 消息标题 否 [2]
    messageContent String 消息内容 否 [3]
    businessCode String 业务编码 是 [4]
    areaCode String 行政区划编码 是 [5]
    sendType String 发送类型 是 [6]
    messageType String 消息类型 否 [7]
    templateCode String 模板标识 否 [8]
    isGroupSend Boolean 是否分组发 否 [9]
    tagId String 分组标签 否 [10]
    isToAll String 是否发送所有人 否 [11]
    isSameChannelContent Boolean 渠道发送内容是否一致 否 [12]
    isSameBatchContent Boolean 批量发送内容是否一致 否 [13]
    file File 文件表单名称 否 [14]
    mediaIds String 素材ID 否 [15]
    fileUrls String 文件url 否 [16]
    isStore String 是否存储消息内容 否 [17]
    isEncrypt String 是否对消息内容加密 否 [18]
    extendField String 扩展字段 否 [19]
    items JSON 消息子项 否(详细参见item参数)[20]

    参数说明

    1. 发送系统编码：接入系统注册时输入编码每个系统的唯一标识，详细查询：接入管理 - 接入系统管理

    2. 消息标题：分组发送和群发时必须参入, 批量发送（一次发送多个人）时批量发送内容一致必须传入，批量发送内容不一致子项消息标题必须传入
      图文消息参考对应渠道格式要求
      同时发送多个渠道消息标题不一致标题格式为： 

    [
  {sendType: '0', messageTitle: '站内消息发送标题'}, 
  {sendType: '1', messageTitle: '邮件消息发送标题'}
]

    1. 消息内容：分组发送和群发时必须参入, 批量发送（一次发送多个人）时批量发送内容一致必须传入，批量发送内容不一致子项消息内容必须传入
      图文消息参考对应渠道格式要求
      同时发送多个渠道消息内容不一致内容格式为： 

    [
  {sendType: '0', messageContent: '站内消息发送内容'},
  {sendType: '1', messageContent: '邮件消息发送内容'}
]

    1. 业务编码: 调用消息平台所需要传入的业务编码，可用作标志业务名称、业务板块名称, 全局唯一。详细查询：配置管理 - 业务编码管理

    2. 行政区划编码: 调用消息平台所需要传入的行政区划编码，可用作标志所属行政区划，全局唯一。详细查询：管理系统 - 系统管理 - 字典管理，字典类型为“AREACODE”

    3. 发送类型: 可用的发送类型有0：站内，1：邮件，2：微信公众号，3：短信，4：APP，5：支付宝生活号。发送多个渠道用‘,’隔开

    4. 消息类型: 具体消息内容类型如文本、图片、音频等具体参考微信公众号、支付生活号消息类型标识

    5. 模板标识: 短信类型必须传入
      同时发送多个渠道模板标识格式为： 

    [
    {sendType: '0', templateCode: 'website_code'}, 
    {sendType: '1', templateCode: 'email_code'}
]

    1. 是否分组发送: 标识微信公众号、支付生活号是否需要分组发送

    2. 分组标签: 分组发送为true时，必须传入

    3. 是否发送所有人: 标识微信公众号、支付生活号是否需要群发，群发则直接调用对应群发接口

    4. 渠道发送内容是否一致: 多渠道发送时，标识各渠道发送内容是否一致，不一致参考对应消息标题和消息内容格式。默认和不传标识内容不一致

    5. 批量发送内容是否一致: 批量发送时（一次发送多个人），标识所有人内容是否一致，不一致子项必须传入消息标题和消息内容。默认和不传标识内容一致

    6. 文件表单名称: 上传文件时文件表单名称，有且必须为file。优先级:文件 > 素材ID > 文件url

    7. 素材ID: 在消息平台上获取文件的ID，多个素材用‘,’隔开。具体参考：素材管理

    8. 文件url: 获取文件的方式直接为文件访问地址

    9. 是否存储消息内容: 是否保存消息发送内容，传入fasle则不保存，默认为true保存消息发送内容

    10. 是否对消息内容加密: 若需要对消息内容加密，接入系统管理必须上传系统公钥证书（RSA 1024位），以用于对消息内容加密。默认为fasle不加密

    11. 扩展字段: 用来在原来设计的字段不足以满足当前渠道或系统调用参数时，传入某些系统或渠道特定的参数，格式为json字符串

    12. 消息子项: 分组发送和群发时不必传入，其他情况必须传入

    • 子体参数（items)
    请求参数 参数类型 参数说明 是否必选
    receiveUserid String 接收人员ID 是[31]
    rsystemCode String 接收系统编码 否[32]
    messageTitle String 消息标题 否[33]
    messageContent String 消息内容 否[34]
    messageType String 消息类型 否[35]
    templateCode String 模板标识 否[36]
    registrationId String app推送id 否[37]
    openId String 微信openId 否[38]
    email String 邮箱账号 否[39]
    phone String 电话号码 否[40]
    toUserId String 支付宝生活号Id 否[41]
    recipientType String 邮件发送接收人类型 否[42]
    extendField String 扩展字段 否[43]

    参数说明

    1. 接收人员ID：发送人员渠道信息未传入时，查询人员是否绑定了渠道信息如email、电话号码等查询不到则不发送

    2. 接收系统编码：接入系统注册时输入编码每个系统的唯一标识，详细查询：接入管理 - 接入系统管理。渠道类型为站内时必须传入

    3. 消息标题：批量发送内容不一致时传入
      图文消息参考对应渠道格式要求
      同时发送多个渠道消息标题不一致标题格式为： 

    [
   {sendType: '0', messageTitle: '站内消息发送标题'}, 
   {sendType: '1', messageTitle: '邮件消息发送标题'}
]

    1. 消息内容：批量发送内容不一致时传入
      图文消息参考对应渠道格式要求
      同时发送多个渠道消息标题不一致内容格式为： 

    [
  {sendType: '0', messageContent: '站内消息发送内容'}, 
  {sendType: '1', messageContent: '邮件消息发送内容'}
]

    1. 消息类型: 具体消息内容类型如文本、图片、音频等具体参考微信公众号、支付生活号消息类型标识。批量发送内容不一致时传入

    2. 模板标识: 批量发送内容不一致时传入，短信类型必须传入
      同时发送多个渠道模板标识格式为： 

    [
  {sendType: '0', templateCode: 'website_code'}, 
  {sendType: '1', templateCode: 'email_code'}
]

    1. app推送id: 推送APP，未绑定用户信息时必须传入

    2. 微信openId: 推送微信公众号，未绑定用户信息时必须传入

    3. 邮箱账号: 发送邮件，未绑定用户信息时必须传入

    4. 电话号码: 发送短信，未绑定用户信息时必须传入

    5. 支付宝生活号Id: 推送支付宝生活号消息，未绑定用户信息时必须传入

    6. 邮件发送接收人类型: "1"为收件人，"2"为抄送人，"3"为密送人，默认是收件人

    7. 扩展字段: 用来在原来设计的字段不足以满足当前渠道或系统调用参数时，传入某些系统或渠道特定的参数，格式为json字符串

    # 3. 返回参数

    返回参数 参数类型 参数说明
    success Boolean 请求是否成功，true成功、false失败
    code String 状态码
    msg String 状态信息
    bizId String 回执ID可根据该ID中查询具体的发送状态
    requestId String 请求ID全局唯一

    • 返回参数示例
        {
      "msg": "短信渠道消息发送完成",
      "code": "200",
      "requestId": "831720c6-96a1-49b2-9a4a-16a0c8f6ea4e",
      "success": true,
      "bizId": "f60cac3a5a83467ba6e33e693250edb1"
    }
