# 邮件渠道消息发送说明

# 1. 功能概述

  • 消息服务平台邮件发送，为接入系统提供发送邮件的功能
  • 支持单发消息
  • 支持批量发送不同内容消息
    • 向指定的一批用户发送不同内容的消息
  • 支持批量发送同一消息
    • 向指定的一批用户发送同一条消息，邮件接收用户可看到邮件接收人类型为收件人和抄送人的其他用户

# 2. 接入流程

  1. 新增接入系统，若其他渠道已新增不必新增，新增时必须选定需要发送的渠道，没有选择则没有该渠道发送权限
  2. 新增邮件配置，"渠道管理-电子邮件配置"，配置发送消息的邮箱
  3. 系统绑定邮件配置，"接入管理-系统绑定"，若未绑定邮件配置，则查询默认配置，都没有则不能发送邮件
  4. 按照消息发送api提供主体参数和子体参数中所需内容，再调用接口即可

# 3. 接口说明

  • 邮件发送统一调用消息发送接口，消息发送类型（sendType）为 “1”
  • 邮件发送无群发(isToAll)、分组发送(isGroupSend)，指定无效，因此子体参数必传

# 4. 参数说明:

  • 主体参数
    • isSameBatchContent
      • 默认为true
      • 批量发送同一条消息时可不传此参数或者传入true，批量发送不同内容消息时必需传入false
      • isSameBatchContent为true时，消息的title和content放入主体参数中
      • isSameBatchContent为false时，消息的title和content放入子体参数中
  • 子体参数
    • receiveUserid
      • 与items参数中的email，必须传入至少一项
      • receiveUserid有值传入时，请确保传入的用wb户绑定了email地址，否则消息将无法发送至该用户邮箱
    • email
      • 传入接收用户的电子邮箱地址
      • 与items参数中的receiveUserid，必须传入至少一项
    • recipientType
      • 邮件接收人类型，"1"为收件人，"2"为抄送人，"3"为密送人，默认是收件人
      • 批量发送同一封邮件时，所有用户可看见其他邮件接收人类型为收件人和抄送人的用户
      • 邮件接收人类型为密送人的用户对其他所有用户不可见

      "sendName": "张三",
      "ssystemCode": "CZ",

      "sendType": "1",
      "businessCode": "email_send_bussiness",
      "areaCode" : "512000",
      "messageTitle":"邮件渠道发送",
      "messageContent":"邮件渠道发送测试"
      "items": [
        {
          "email":"1244203665@qq.com",
        }
      ]
    }
