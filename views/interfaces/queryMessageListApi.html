# 消息列表查询接口文档

# 1. 接口描述

  • 接口名称 消息列表查询接口
  • 接口说明 该接口可分页查询当前指定用户发送消息和接收消息相关信息，可以通过消息状态、发送类型、是否已读进行筛选 注：接收消息情况只支持查询站内，消息状态为成功（200），只能通过是否已读进行筛选
  • 接口地址

调用示例

http://localhost:8081/git/api/message/queryMessage
（消息平台地址 + api/message/queryMessage）

  • 接口编码
    UTF-8
  • 支持协议 HTTP(POST)

# 2. 请求参数

请求参数 参数类型 参数说明 是否必选
queryType String 查询类型，分为发送消息和接收消息,参考查询类型 是
systemCode String 当前用户系统编码 是
userId String 当前用户id 是
state String 消息状态，200为成功，其他值均为失败，可参考系统配置管理-状态码管理 否
sendType String 发送类型，参考发送类型 否
read String 是否已读，参考是否已读 否
pageNumber String 当前页码，不传则默认为1 否
pageSize String 每页显示条数，不传默认为5，最多每页显示20条 否

# 3. 返回参数

返回参数 参数类型 参数说明
code String 状态码
requestId String 请求ID全局唯一
serviceSuccess Boolean 业务是否执行成功
errors JSON数组 业务执行失败信息
data JSON 结果集

  • data 参数说明
返回参数 参数类型 参数说明
pageBean JSON 分页相关信息

  • pageBean 参数说明
返回参数 参数类型 参数说明
pageNum String 当前页码
pageSize String 每页数据条数
currentSize String 当前每页条数
total String 总条数
pages String 总页数
list JSON数组 数据内容

  • list 参数说明
返回参数 参数类型 参数说明
id String 消息id
receiveUserId String 接收用户id
rsystemCode String 接收系统编码
sendUserId String 发送用户id
ssystemCode String 发送系统编码
sendType String 发送类型,参考发送类型
messageType String 消息类型
templateCode String 模板编码，如果当前消息通过模板发送，则该字段有值
messageTitle String 消息标题
messageContent String 消息内容
read String 是否已读，参考是否已读
registrationId String app推送id，当为app消息时，可能该字段有值
openId String 微信推送id，当为微信公众号消息时，可能该字段有值
email String 邮箱地址，当为邮件消息时该字段有值
recipientType String 邮件接收人类型，参考接收人类型
toUserId String 支付宝推送id，当为支付宝生活号消息时，该字段可能有值
state String 消息状态，200为成功，其他值均为失败，可参考系统配置管理-状态码管理
stateInfo String 消息状态信息
createDate String 消息创建时间
updateDate String 消息更新时间

# 4. 调用示例

  • 该接口可直接使用框架的分页进行访问,也可以手动发起请求，自己解析参数
  • 请求参数
    • 查询发送消息列表
      • 示例：
          params = {
           	"queryType": "1",
           	"systemCode": "PORTAL",
           	"userId": "1",
           	"sendType": "0",
           	"state": "200",
           	"read": "1",
           	"pageSize": "10",
           	"pageNumber": "2"
           }
    • 查询接收消息列表
      • 示例：
          params = {
            	"queryType": "2",
            	"systemCode": "PORTAL",
            	"userId": "1",
            	"read": "1",
            	"pageSize": "10",
            	"pageNumber": "2"
            }
  • 框架调用示例：
     // 标签：
  <ta-pagination style="float: right; margin-top: 10px;" 
      :defaultPageSize="5" 
      :url="api/message/queryMessage" 
      :params = "fnGetParam" 
      :dataSource.sync="tableData" 
      ref="gridPager"/>
  
  // 获得参数方法：    
  fnGetParam(){
      let params = {};
      params.systemCode = "PORTAL";
      params.userId = parent.indexTool.getUserInfo().userId;
      params.queryType = "2";
      return params;
  }
  
  // 执行请求方法：
  this.$refs.gridPager.loadData();
